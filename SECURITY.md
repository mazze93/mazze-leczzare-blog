# Security Policy

## Supported Versions

Security fixes are applied to the latest code on the `main` branch.

| Version / Branch | Supported |
| ---------------- | --------- |
| `main`           | ✅         |
| Older releases   | ❌         |

## Reporting a Vulnerability

If you believe you’ve found a security vulnerability, please **do not open a public issue**.

Instead, report it using one of the following:

### 1. GitHub Private Vulnerability Reporting (Preferred)

Go to: **Security → Advisories → Report a vulnerability** in this repository.

### 2. Email

Send details to: **[security@mazzeleczzare.com](mailto:security@mazzeleczzare.com)**

Please include:

* A clear description of the issue and potential impact
* Steps to reproduce (proof-of-concept if available)
* Affected component(s) or files
* Any suggested mitigation

## What to Expect

* **Acknowledgement within 3 business days**
* Status updates at least every 7 days until resolution
* Coordinated disclosure after a fix is available
* Credit in release notes/advisories (if desired)

We request a reasonable remediation window before public disclosure.
